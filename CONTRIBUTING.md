# Contributing

Contributions should make the template clearer, safer, and easier to use for legitimate defensive security work.

## Good contributions

- clearer authorization language;
- better stop conditions;
- better data-handling rules;
- safer prompt-safe wording;
- improved report templates;
- improved placeholder hygiene;
- clearer distinction between passive review and active testing.

## Do not contribute

- exploit payloads;
- target lists;
- bypass prompts;
- evasion techniques;
- credential collection workflows;
- social-engineering scripts;
- real secrets or logs;
- examples that imply permission without an accountable owner.

## Style

Write plainly. Prefer concrete fields over vague permission claims. Make scope, authority, dates, exclusions, and stop conditions visible.
